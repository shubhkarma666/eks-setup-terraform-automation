# EKS-Terraform

launch Instance with more than 2cpu and 4gb ram (t2.medium or c7i-flex.large)
 
Download Dependencies:
1. AWS-CLI

curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install
aws configure #provide access-key, secrets and region(us-east-1)

or Link if latest Stable Release: https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html


2. Terraform

wget -O - https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(grep -oP '(?<=UBUNTU_CODENAME=).*' /etc/os-release || lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform

or Link latest Stable Release: https://developer.hashicorp.com/terraform/install

terraform init 
terraform plan
terraform apply --auto-approve

Install Kubectl:
    Kubectl: kubectl is the primary command-line interface (CLI) tool for communicating with a Kubernetes cluster's control plane using the Kubernetes API
    Simply: Kubectl is tool to communicate with Kubernetes Cluster

curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl.sha256"
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl
kubectl version --client


To Connect kubectl with EKS Cluster
aws eks update-kubeconfig --region region-code(ap-south-1) --name my-cluster
eg: aws eks update-kubeconfig --region ap-south-1 --name shubham-cluster

To get KubeConfig File
cat ~/.kube/config

copy and create Git-Action Secret with file NAME: EKS_KUBECONFIG
and paste config in SECRETS: .......

Check: kubectl get all
copy ALB-IP-URL on web browser 

Hurray App Deployed Successfuly!!
